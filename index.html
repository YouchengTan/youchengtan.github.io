Summary

  We are building an interactive, stylized simulation of a soft-body “jelly warrior” using a spring-mass model. Our goal is to create a visually satisfying experience that showcases squishy deformation, wobble animations, and fun jelly-like shading. The scene will allow the user to rotate the camera to appreciate the dynamic elasticity in real time. Inspired by soft-body demos in animations and games, this project aims to be engaging, polished, and technically feasible.

Problem Description

  Soft-body deformation is widely used in games and simulations to model characters, objects, or materials that behave elastically (like jelly or rubber). Unlike rigid-body simulations, soft-body models must account for internal structure, elasticity, and response to external forces. While more physically accurate simulations use finite element methods (FEM), we aim to use a spring-mass system to achieve real-time visual fidelity with less computational overhead. The core challenge is creating realistic yet stable deformation behavior, especially under animation or collisions. We'll implement a spring-mass soft-body framework, design a 3D jelly warrior model, and render it with shaders that give a translucent or subsurface-scattering-like appearance. This problem is both educational and visually impactful. Our approach is inspired by real-time visual references like:
Steve Final Smash – Softbody Simulation
Jelly Simulation Short

There are a few main challenges:

  1. Accurate force simulation between the triangles on the mesh.
  2. Gravity simulation and initial conditions (forces, velocity)
  3. Collision detection between different bodies

Goals and Deliverables

  We aim to answer: Can a spring-mass-based simulation produce visually convincing soft-body deformation in real time with a stylized shader? Success will be measured by visual responsiveness, stability, and perceived realism in the demo video.
Here we lay out the goals we should achieve as a base line for this project:

  • A working spring-mass soft-body simulation for a jelly character
  • Elastic deformation during idle wobble and a punch animation
  • Simple ground-plane collision response
  • A stylized shader for a translucent, jelly-like effect
  • Interactive camera controls (rotate, zoom)
  • A polished demo video and screenshots of key moments
  • Full write-up with explanation of our implementation and results

Here we lay out the goals we hope to achieve for this project:
  • A toggle to show the internal spring network or deformation heatmap
  • Slightly enhanced jelly shading (e.g., fake subsurface or Fresnel effect)
  • A decorative background scene (e.g., simple dojo or floating platform)
  • A second jelly object with limited interaction (if time permits)
